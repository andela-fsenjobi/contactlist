    Introduction

    Welcome to the Contactlist API! You can use our API to access Contactlist API endpoints. With this you can keep track of transactions and customers.

    I run a small business and I have been looking at ways to use technology to make record keeping easy for myself. I came up with this API to solve this problem.

    You can make use of the available API endpoints for similar problems.

    Registration

    To register, type this in your console:

    curl -X POST \
  "https://my-contactlist.herokuapp.com/api/users?\
  user[email]=my@email.com&user[password]=1qw23er45t"

    The response to the above should be of this nature:

    {
  "id": 132,
  "email": "me@email.com",
  "created_at": "1qw23er45ty67uu8r.9iwi92820900ei84irri93"
}

    The API endpoints are only available to registered user. In order to have access, you would have to register.

    The email field is validated and thus, will return ‘422 - Unprocessable Entity error’

    The password field requires minimum of 6 characters

    Authentication

    To authorize, use this:

    curl -X POST \
  "https://my-contactlist.herokuapp.com/api/auth/login?\
  session[email]=my@email.com&session[password]=1qw23er45t"

    The response to the above should be of this nature:

    {
  "email": "me@email.com",
  "auth_token": "1qw23er45ty67uu8r.9iwi92820900ei84irri93",
  "message": "Your have been logged in"
}

    Once registered, you would have to login. If your authentication was successful, you would get a token, with which you can access all other endpoints.

    Take note of your token. Without it, you would not be able to access any other endpoint asides registration and login

    Customers

    HTTP Method Endpoint Use
    POST /customer Create a customer
    GET /customers/ Show list of all customers
    GET /customers/1 Show details of a single customer
    PATCH /customers/1 Update details of a single customer
    DELETE /customers/1 Delete customer from database
    Remember — all these endpoints require Authentication

    Some of the above features are demonstrated below:

    Create

    To create a customer, type this:

    curl -X POST -H \
  "Authorization: <insert your token here>" \
  "https://my-contactlist.herokuapp.com/api/customers? \
  customer[name]=Femi Adebayo&customer[phone]=08123428749"

    JSON Response

    {
  "id": 1,
  "name": "Femi Adebayo",
  "phone": "08123428749",
  "created_at": "2016-04-31 10:00:00 +0100"
}

    The Customer table has the following fields:

    • Name
    • Phone
    • Referer (optional)

    Requirements:

    • Endpoint: https://my-contactlist.herokuapp.com/api/customers
    • Method: POST
    • Authentication: required
    • Parameters: name, phone, referer (optional)

    All Customers

    To view all customers, use this:

    curl -X GET -H \
  "Authorization: <insert your token here>" \
  "https://my-contactlist.herokuapp.com/api/customers"

    JSON Response

    [
  {
    "id": 1,
    "name": "Femi Adebayo",
    "phone": "08123428749",
    "created_at": "2016-04-31 10:00:00 +0100"
  }
]

    Requirements:

    • Endpoint: https://my-contactlist.herokuapp.com/api/customers
    • Method: GET
    • Authentication: required
    • Parameters: Since the result form this endpoint is paginated, you can specify the page number and number of items per page by supplying additional parameters.
      • Page: You can pass on the required page number to the endpoint. For instance, by appending &page=2 you are requesting for the second page.
      • Limit: By default the number of items per page is 20. You can specify your own limit by passing the parameter limit to the endpoint. &limit=30 will return 30 items for each request
      • Search query (optional): You can search customers by number and name. The search query uses the LIKE so that not just exact matches are returned. All items that match the query are returned. Result from the search is also paginated and can be combined with the page and limit parameters.

    Transactions

    HTTP Method Endpoint Use
    POST /customers/1/transactions Create a new transaction
    GET /customers/1/transactions Show transactions from a particular customer
    GET /transactions Also works to display all transactions but from all customers
    GET /customers/1/transactions/1 Show details of a single transactions
    PATCH /customers/1/transactions/1 Update transactions
    DELETE /customers/1/transactions/1 Delete transactions from database
    Remember — all these endpoints require Authentication

    Some of the above features are demonstrated below:

    Create a Transaction

    To create a transaction, use this:

    curl -X POST -H \
  "Authorization: <insert your token here>" \
  "https://my-contactlist.herokuapp.com/api/customers/1/transactions? \
  transaction[amount]=200&transaction[expiry]=2016-05-29"

    JSON Response

    {
  "id": 1,
  "status": "Paid",
  "amount": -1000,
  "expiry": "2016-05-29",
  "created_at": "2016-04-29T10:30:49.080Z"
}

    The Transaction table has the following fields:

    • Amount
    • Expiry
    • Status

    Requirements:

    • Endpoint: https://my-contactlist.herokuapp.com/api/customers/1/transactions
    • Method: POST
    • Authentication: required

    Stats

    Total

    To view total statistics, use this:

    curl -X GET -H \
  "Authorization: <insert your token here>" \
  "https://my-contactlist.herokuapp.com/api/stats/total"

    JSON Response

    {
  "data": {
    "total_customers": 10,
    "total_transactions": 3,
    "total_gains": -800
  }
}

    This API offers the opportunity to see statistics from all transactions and customers. This endpoint gives a summary of the users all-time statistics. Users can find out:

    • How many customers they have all together
    • How many transactions they have made all together

    • How much profit you have made from all transactions

      • Endpoint: https://my-contactlist.herokuapp.com/api/stats/total
      • Method: GET
      • Authentication: required
